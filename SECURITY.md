# Security Policy

## Reporting a vulnerability

Please report security issues privately — do not open a public issue.

- Email: [security@releases.sh](mailto:security@releases.sh)
- Or use [GitHub private vulnerability reporting](https://github.com/buildinternet/releases/security/advisories/new) on this repository.

The canonical disclosure policy lives at [releases.sh/security](https://releases.sh/security) (machine-readable: [`/.well-known/security.txt`](https://releases.sh/.well-known/security.txt)).

We do not offer a bug bounty or paid reward program.

## Scope

This repository is the source of the hosted service at releases.sh — the API worker, MCP server, web frontend, discovery worker, and webhook delivery. Reports about the hosted service and reports about the code are equally welcome through the channels above. For the CLI, report via the same email or in [buildinternet/releases-cli](https://github.com/buildinternet/releases-cli).
